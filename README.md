# Inception

## Description

This project is a ***hands-on***, complete project designed to learn ***Docker and Docker compose***, as well as, ***web technology architecure*** and the basics of:
* ***nginx with php-fpm***
* ***wordpress***
* ***mariadb***

### Goal

* Create a complete server capable of ***hosting wordpress with a db efficiently*** and securely

### Design choices

* In my project, a Dockerfile sets-up the container with the basic modules and configuration and the entrypoint script handles more detailed set up

### Virtual Machine vs Docker

| | ***Virtual machine*** | ***Docker*** |
| -- | -- | -- |
| ***OS*** | Complete OS | Shares OS kernel with host |
| ***Memory usage*** | High | Low |
| ***Hardware*** | Virtualized | Shared |
| ***Software Isolation*** | Can't access host OS | Blocks access to internet - Host files |

### Secrets vs Environment variables

|| ***Secrets*** | ***Environment variables*** |
|---|---|---|
|***Read***|File in ```/run/secrets``` inside the image|Passed as ```char **``` as the third argument to programs|
|||Stored in terminal environment variables|
|***Write***|Specified at```docker_compose.yaml```, the file is copied|Setted inside ```docker_compose.yaml```|
|||A ```.env``` file can be created to modularize the proccess|
|***Security***|Permissions can be set to files to restrict access|Any program gets access to environment variables|

### Docker Network vs Host Network
* Docker networks in contrast to the Host network, gives the application limited access to ports specified in the docker compose, in addition, it can only communicate with external images or host ports when specified.

## Instructions

* ***Install docker - docker compose - make***, if not installed yet
* Check docker status (```sudo systemctl status | grep docker```) and start it if not active with ```sudo systemctl start docker```
* ***Run the command ```make```*** at the project root
* ***Visit ```localhost```*** at a web browser
* To use a ***custom domain name locally*** add the line ```127.0.0.1 my.custom.domain.com``` to ```/etc/hosts```
* To stop the containers, run ```make down``` at the project root
* To delete all data generated by the project run ```make clean``` at project root
